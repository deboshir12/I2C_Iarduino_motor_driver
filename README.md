# control_motors

Прошивка для STM32G474CEUx, которая принимает простые команды по UART и управляет моторным контроллером по I2C.

## Что делает проект

- поднимает `USART1` на скорости `115200 8N1`;
- принимает строковые команды `pwm` и `rpm`;
- парсит значение как `int16_t`;
- отправляет настройки и команды моторному контроллеру на I2C-адрес `0x09`;
- отвечает в UART статусом выполнения команды.

## Аппаратная конфигурация

Проект сгенерирован для STM32CubeIDE из `control_motors.ioc`.

- MCU: `STM32G474CEUx`, корпус `UFQFPN48`.
- Clock: HSI 16 MHz, PLL не используется.
- I2C: `I2C1`.
- UART: `USART1`, `115200 8N1`.

Назначение выводов:

| Пин | Назначение |
| --- | --- |
| `PA15` | `I2C1_SCL` |
| `PB7` | `I2C1_SDA` |
| `PC4` | `USART1_TX` |
| `PA10` | `USART1_RX` |

## UART-протокол

Команды завершаются `\r`, `\n` или `\r\n`. Разделителем между командой и значением может быть пробел, табуляция или `=`.

Поддерживаемые команды:

```text
pwm <value>
pwm=<value>
rpm <value>
rpm=<value>
```

`value` должен помещаться в диапазон `int16_t`: от `-32768` до `32767`.

Примеры:

```text
pwm 120
rpm=1500
pwm -80
```

Ответы:

```text
OK pwm
OK rpm
ERR cmd
ERR pwm st=<hal_status> i2c=<i2c_error>
ERR rpm st=<hal_status> i2c=<i2c_error>
```

## I2C-обмен с моторным контроллером

Настройки находятся в `Core/Inc/motor_i2c.h`, реализация - в `Core/Src/motor_i2c.c`.

Текущие параметры:

- адрес устройства: `0x09`;
- таймаут I2C-операций: `10` ms;
- число магнитов по умолчанию: `7`;
- редуктор по умолчанию: `1450`.

Используемые регистры:

| Регистр | Адрес | Назначение |
| --- | --- | --- |
| `MOTOR_NUM_MAGNETS_REG` | `0x11` | число магнитов |
| `MOTOR_REDUCER_REG` | `0x12` | коэффициент редуктора, 24-bit LE |
| `MOTOR_SET_PWM_REG` | `0x15` | установка PWM, `int16_t` LE |
| `MOTOR_SET_RPM_REG` | `0x17` | установка RPM, `int16_t` LE |

При старте `main.c` вызывает `Motor_Config()`, которая записывает число магнитов и редуктор. Затем основной цикл обрабатывает UART-команды и вызывает `Motor_SetPwm()` или `Motor_SetRpm()`.

## Структура проекта

- `Core/Src/main.c` - инициализация HAL, I2C, UART и основной цикл.
- `Core/Src/uart_commands.c` - прием UART в interrupt-режиме, буферизация строки и парсер команд.
- `Core/Inc/uart_commands.h` - типы и публичный API UART-команд.
- `Core/Src/motor_i2c.c` - запись команд и настроек в моторный контроллер.
- `Core/Inc/motor_i2c.h` - I2C-адрес, регистры и параметры по умолчанию.
- `control_motors.ioc` - конфигурация STM32CubeMX/STM32CubeIDE.

## Сборка и прошивка

1. Открыть workspace в STM32CubeIDE.
2. Импортировать проект `control_motors` как существующий STM32CubeIDE-проект.
3. Собрать конфигурацию `Debug`.
4. Прошить целевую плату через созданный launch configuration или стандартный STM32CubeIDE debug/run flow.

Сгенерированные артефакты `Debug/`, `.settings/`, `.cproject`, `.mxproject`, `.project`, `.launch` и linker scripts игнорируются через `.gitignore` по тому же шаблону, что и в соседних проектах.
